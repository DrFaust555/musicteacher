# Contributing to MusicTeacher

Thank you for your interest. MusicTeacher is transitioning from closed-source to open-source as part of an NLnet Foundation NGI Zero Commons funding application (see [README](README.md) for context). **The source code is not yet published**, so we cannot yet accept code contributions — but we can use other forms of help, and we are setting up the infrastructure now so that contribution can begin smoothly once the codebase is open.

## How you can help right now

- **Star and watch this repository.** This is the single most useful low-cost action — community signal materially helps grant applications, and once the source lands you will get a notification.
- **Open an issue.** We are especially interested in:
  - *I'm a music teacher and here is a feature that would make this useful in my lessons.* Pedagogical input is the rarest and most valuable signal.
  - *I'm a student or self-learner and here is the workflow I wish existed.*
  - *I work on / maintain an open-source music or audio project and see a possible collaboration.*
- **Get in touch about consulting work.** If you teach piano, guitar, or voice at conservatory level and would be interested in being part of the grant-funded curriculum work, please reach out: musicneutrino@gmail.com. Hours are paid; we are looking for 2–3 specialists.
- **Spread the word** to anyone in music education, open-source music tools (MuseScore, Frescobaldi, Lomse, JUCE), or EU funding ecosystems who might find the project interesting.

## How you can help once the source is open

Once the licence audit completes and the source is published (target: October 2026 — see Roadmap in README), normal open-source contribution practices apply:

- **Bug reports** — open an issue with reproduction steps, your OS, the score file (if relevant), and the version of MusicTeacher
- **Pull requests** — small focused changes are easier to review than large ones. Discuss larger work in an issue first
- **Documentation, translations, lesson library entries** — these are first-class contributions, not second-tier

A more detailed contribution guide (style, test expectations, signed commits) will be added with the first source release.

## Project governance

For the duration of the NLnet-funded work (initial twelve months from grant start), the maintainer is the project lead and final decision-maker on direction. As contributors join, governance will move toward a more collaborative model — documented in this file as it evolves.

## Code of conduct

We follow the [Contributor Covenant](https://www.contributor-covenant.org/) for all interactions in this project's spaces (issues, discussions, pull requests). Brief version: be kind, assume good faith, criticize ideas not people. The full text will be added with the first source release.
